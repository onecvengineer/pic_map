# Documentation Discipline

每次功能提交需要同步更新文档。

## 必须更新

- 新增 API：更新 `docs/api.md`。
- 修改数据结构：更新 `docs/schema.md`。
- 修改部署环境变量：更新 `README.md`、`.env.example`、`docs/operations.md`。
- 修改导入、备份、恢复或迁移流程：更新 `docs/operations.md`。
- 修改产品行为：更新 `docs/product.md`。

## 提交说明

提交或 PR 描述中必须列出：

- 代码改动
- 文档改动

如果不涉及文档，明确写“不涉及文档”。
